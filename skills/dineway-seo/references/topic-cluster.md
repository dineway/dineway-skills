# Topic Cluster Workflow

Use this to plan hub/spoke content architecture from a seed topic.

## Method

1. Expand the seed into query variants:
   - head terms;
   - modifiers;
   - local modifiers;
   - comparison and alternative terms;
   - intent-stage questions;
   - seasonal/event variants.
2. Use DataForSEO keyword suggestions and SERP results when live data is needed.
3. Cluster by SERP overlap and search intent, not only wording similarity.
4. Choose a hub page only when it can answer broad intent and naturally link to spokes.
5. Define spokes around distinct intent, not duplicated thin pages.

## Dineway Rules

- Map each planned page to an existing Dineway collection or a proposed collection/field change.
- Keep restaurants/local businesses practical: menu pages, occasion pages, neighborhood/location pages, private dining, catering, reviews, gallery, and editorial updates.
- Avoid doorway pages. A local or keyword page needs unique source-backed value.
- No local HTML artifacts or auto-generated file maps are part of v1.

## Output

Return:

- `Seed`
- `Cluster Map`
- `Hub Recommendation`
- `Spoke Pages`
- `Internal Link Matrix`
- `Dineway Content Targets`
- `Provider Evidence`
- `Skipped Terms`
- `Measurement Plan`
