# FLOW Framework

FLOW is the operating model for evidence-backed SEO work:

- `Find`: discover demand, competitors, gaps, crawl/indexing issues, and local/AI visibility gaps.
- `Leverage`: use existing strengths such as reviews, menu data, location authority, images, internal links, and high-performing pages.
- `Optimize`: improve metadata, headings, content structure, schema, internal links, image SEO, page speed signals, and crawlability.
- `Win`: create or update pages that can compete for priority intent.
- `Local`: strengthen restaurant/local business facts, maps visibility, neighborhood relevance, reviews, and visit guidance.

## Rules

- Attribute recommendations to the relevant FLOW stage.
- Each recommendation needs evidence, impact, effort, and Dineway write target.
- Local work must be grounded in real place data and customer-facing value.
- AI/answer-engine visibility work must focus on crawlable, source-backed, quotable pages and clear entity facts.
